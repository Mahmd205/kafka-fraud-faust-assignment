# Real-Time Credit Card Fraud Detection Using Apache Kafka and Faust

## 1. Project Overview

This project implements a complete real-time streaming machine learning pipeline using **Apache Kafka** and **Faust**.

The application reads credit card transaction rows from a public dataset, streams them as live JSON events into a Kafka topic, performs real-time fraud prediction using a pre-trained machine learning model, and publishes the prediction results to an output Kafka topic.

The full pipeline is:

```text
creditcard.csv → Producer → raw-data topic → Faust Streams Processor → predictions topic → Output Consumer
```

## 2. Chosen Assignment Options

| Requirement | Choice |
|---|---|
| Programming language | Python |
| Streams API | Faust (`faust-streaming`) |
| Dataset | Credit Card Fraud Detection dataset |
| ML task | Binary classification: fraudulent vs. normal transaction |
| ML model | Logistic Regression with class balancing |
| Kafka setup | Local Kafka using Docker |

## 3. Dataset

Dataset: **Credit Card Fraud Detection**

Source: Kaggle dataset by Machine Learning Group - ULB.

Download the dataset manually from Kaggle and place the CSV file here:

```text
data/creditcard.csv
```

Expected columns:

```text
Time, V1, V2, ..., V28, Amount, Class
```

Where:

- `Class = 0`: normal transaction
- `Class = 1`: fraudulent transaction

> Note: The dataset is not included in this repository because Kaggle datasets often require account authentication.

## 5. Installation

Recommended Python version: **Python 3.10 or 3.11**.

### Step 1: Create and activate a virtual environment

Windows PowerShell:

```bash
python -m venv .venv
.venv\Scripts\activate
```

macOS/Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### Step 2: Install dependencies

```bash
pip install -r requirements.txt
```

## 6. Start Kafka

Start Kafka locally using Docker:

```bash
docker compose up -d
```

Check that Kafka is running:

```bash
docker ps
```

## 7. Create Kafka Topics

Run:

```bash
python src/create_topics.py
```

This creates:

- `raw-data`
- `predictions`

## 8. Train the ML Model Offline

After placing `creditcard.csv` inside the `data/` folder, run:

```bash
python src/train_model.py
```

This script:

1. Loads the full dataset.
2. Splits the data into training and testing sets.
3. Trains a Logistic Regression model.
4. Reports Accuracy and F1-score.
5. Saves the trained model to:

```text
models/fraud_model.joblib
```

Example output format:

```text
Model performance
Accuracy: 0.9991
F1-score: 0.8123
Model saved to models/fraud_model.joblib
```

Your exact values may differ depending on library versions and random split.

## 9. Run the Real-Time Pipeline

Open **three terminals** side by side.

### Terminal 1: Start the Faust Streams Processor

```bash
faust -A src.app worker -l info
```

This processor consumes from `raw-data`, loads the saved ML model, predicts fraud, and publishes results to `predictions`.

### Terminal 2: Start the Output Consumer

```bash
python src/consumer.py
```

This prints prediction results live.

### Terminal 3: Start the Producer

```bash
python src/producer.py --rate 1
```

This reads the dataset row by row and publishes about one transaction per second to the `raw-data` topic.

## 10. Expected Live Output

The output consumer will print messages like:

```text
Transaction tx_000001 | Prediction: NORMAL | Fraud probability: 0.0342 | Actual class: 0
Transaction tx_000002 | Prediction: FRAUD  | Fraud probability: 0.9127 | Actual class: 1
```

## 11. Model Performance Reporting

```text
Accuracy: 0.9755
F1-score: 0.114358
```

The F1-score is especially important because the dataset is highly imbalanced. A high accuracy alone can be misleading when fraudulent transactions are rare.

## 12. Notes

- The Streams API requirement is satisfied by `src/app.py`, which uses Faust agents.
- The ML model is trained offline using `src/train_model.py`.
- The producer replays the dataset as live events at approximately 1 row per second.
- The consumer reads the prediction topic and prints each result in real time.

```bash
docker compose down
```
